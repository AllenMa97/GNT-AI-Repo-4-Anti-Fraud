3. 核心洞察是什么？

### 2.2 电信诈骗侦查的核心洞察

**观察到的现象**：

公安侦查人员在分析诈骗案件时，存在**"信息分工"**模式：

| 任务 | 需要的信息 | 不需要的信息 |
|------|-----------|-------------|
| **诈骗类型分类** | 被害人陈述、关键词、聊天模式 | 身份证号、银行卡号 |
| **资金追踪** | 交易金额、时间戳、账户 | 聊天内容细节 |
| **嫌疑人画像** | 行为模式、通讯频率、关系网络 | 具体地理位置、声音 |
| **证据链构建** | 时间线、因果关系 | 个人身份信息 |

**核心洞察（Task Information Selectivity in Investigation）**：

> "电信诈骗侦查任务的完成**依赖于特定的信息子集**，而非全部案件信息。这个信息依赖关系与公安SOP（标准作业程序）高度吻合——SOP的每一步只依赖上游的必要信息，而非全部历史。"

**类比ResNet**：
- ResNet：恒等映射是深层网络的优化捷径
- 我们的洞察：**信息最小化是LLM推理的隐私捷径**

### 2.3 公安SOP的本质

公安SOP（标准作业程序）本质上是一个**信息依赖图**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        公安侦查SOP = 信息依赖图                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   Step 1: 受案登记                                                    │
│   需要：报案人陈述、联系方式                                           │
│   不需要：详细聊天记录、交易流水                                        │
│                         ↓                                             │
│   Step 2: 初步分类（刷单/投资/杀猪盘）                                 │
│   需要：关键词、通讯模式、金额范围                                      │
│   不需要：具体账户号、时间戳                                           │
│                         ↓                                             │
│   Step 3: 紧急性评估                                                  │
│   需要：涉及金额、紧急程度、是否正在被骗                                │
│   不需要：详细话术分析                                                 │
│                         ↓                                             │
│   Step 4: 资金追踪                                                    │
│   需要：交易流水、金额、时间戳                                         │
│   不需要：聊天内容详细分析                                             │
│                         ↓                                             │
│   Step 5: 嫌疑人研判                                                  │
│   需要：行为模式、通讯网络、资金流向                                    │
│   不需要：被害人详细身份信息                                           │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**写作要点（150词摘要）**：

> Telecom fraud causes billions in losses annually. Public security bureaus increasingly deploy LLMs for case analysis, but face privacy-quality trade-off. We observe that in criminal investigation workflows, different analytical tasks exhibit strong information selectivity — fraud classification needs keywords, not identities; fund tracing needs amounts, not chat content. This is aligned with the SOP (Standard Operating Procedure) used by police: each investigation step depends only on necessary upstream information. We propose an SOP-aware LLM framework that filters information based on task requirements while respecting investigation protocols. Experiments on our constructed CTFD dataset and real-world police data validate our approach, achieving 97% quality with 62% information reduction.

#### Section 3: Investigation Information Selectivity（核心洞察）

**这是论文的灵魂！**

**3.1 观察**

在电信诈骗案件中，我们观察到：

| 任务 | 关键信息 | 可省略信息 |
|------|---------|-----------|
| 诈骗分类 | 被害人陈述、关键词、通讯模式 | 身份证号、详细住址 |
| 资金追踪 | 金额、时间戳、账户尾号 | 账户持有人姓名 |
| APP分析 | APP名称、界面描述 | 下载记录详情 |
| 嫌疑人画像 | 行为模式、通讯频率 | 声音、面部信息 |

**3.2 形式化**

定义**侦查信息集**$I = \{I_1, I_2, ..., I_k\}$和**侦查任务集**$T = \{T_1, T_2, ..., T_m\}$。

**观察1（侦查信息选择性）**：
$$\forall T_j, \exists I_{sel} \subseteq I : Q(I_{sel}, T_j) \geq Q(I, T_j) - \epsilon$$

**3.3 SOP作为信息依赖图**

公安SOP定义了任务间的信息依赖关系：
$$T_j \leftarrow \{T_i | \text{SOP定义了}T_j\text{依赖}T_i\}$$

这个依赖图与信息选择性高度吻合！

#### Section 4: SOP-Aware LLM Framework

**4.1 框架设计**

```
┌─────────────────────────────────────────────────────────────────────┐
│                   SOP-Aware LLM Framework                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   Case Data D ──→ SOP Parser ──→ Task Analyzer                      │
│                              ↓                                       │
│                      当前任务T_j需要的信息I_j                        │
│                              ↓                                       │
│                      隐私风险评估                                     │
│                      I(I_j; PII) ≤ δ?                              │
│                              ↓                                       │
│                      信息过滤F_j(D) ──→ LLM                         │
│                              ↓                                       │
│                      任务输出Y_j                                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```
