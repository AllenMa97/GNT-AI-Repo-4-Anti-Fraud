# English Paper Outline (AAAI/ACL Format)

## 1 Introduction

### Motivation
Deploying large language models (LLMs) in sensitive domains (healthcare, legal, finance, public security) faces a fundamental tension:
- **Privacy regulations** (PIPL, GDPR) prohibit sending raw sensitive data to external APIs
- **Anonymization** removes both noise AND signal, degrading task quality

### Core Observation
> **Task Information Selectivity**: Different tasks require different information subsets.

For example:
- Fraud classification needs keywords, communication patterns
- Fund tracing needs transaction amounts, timestamps, account numbers
- Suspect profiling needs behavioral metadata

### Key Insight
> **Information Minimization Principle (IMP)**: For each task $T_i$, there exists a *Minimum Sufficient Set* $M_i \subseteq D$ such that $Q(M_i, T_i) \approx Q(D, T_i)$ while $|M_i| \ll |D|$.

This insight is analogous to **ResNet's core observation**: "identity mapping is easier to learn than the residual function."

### Framework
We propose **IMFORM**: Information Minimization Framework for LLM deployment.

### Contributions
1. **Task Selection Hypothesis**: A formal observation about task information selectivity
2. **IMP Formalization**: Information-theoretic framework for task-adaptive privacy filtering
3. **IMFORM Architecture**: A general framework with three implementations
4. **Extensive Experiments**: Validated across healthcare, legal, finance, and security domains

## 2 Problem Formulation

### 2.1 Task Information Selectivity

**Definition 1 (Minimum Sufficient Set, MSS)**: For task $T_i$ on data $D$, $M_i \subseteq D$ is an MSS if:
1. $Q(M_i, T_i) \geq Q(D, T_i) - \epsilon$ (quality preservation)
2. $|M_i| \ll |D|$ (information minimization)

### 2.2 Information Minimization Principle

**Hypothesis 1 (IMP)**: For most NLP tasks, there exists an MSS $M_i$ such that:
$$Q(M_i, T_i) \approx Q(D, T_i) \quad \text{and} \quad |M_i| \ll |D|$$

### 2.3 Privacy-Utility Trade-off

**Definition 2 (Privacy Leakage)**: $I(M_i; PII)$ measures the mutual information between selected information and sensitive attributes.

**Hypothesis 2 (Impossible Triangle)**: It is impossible to simultaneously achieve:
- Zero leakage: $I(M_i; PII) = 0$
- Optimal quality: $Q(M_i, T_i) = Q^*(T_i)$
- Baseline efficiency: $C(M_i) \leq C_{baseline}$

## 3 Theoretical Analysis

### 3.1 Why IMP Works: Information-Theoretic Foundation

**Theorem 1 (Task Irrelevance)**: For task $T_i$, there exists a subset $D \setminus M_i$ that is *irrelevant*:
$$I(T_i; D \setminus M_i | M_i) \approx 0$$

*Proof sketch*: By chain rule of mutual information, we can decompose:
$$I(T_i; D) = I(T_i; M_i) + I(T_i; D \setminus M_i | M_i)$$

If the second term is small, $M_i$ contains sufficient information for $T_i$.

### 3.2 Information Compression Bound

**Theorem 2 (Compression Rate)**: For task $T_i$, the minimum compression rate $r_i = |M_i|/|D|$ satisfies:
$$r_i \geq \frac{H(T_i | D \setminus M_i)}{H(D)}$$

This provides a **theoretical lower bound** on how much information can be safely removed.

### 3.3 Multi-Task Information Sharing

**Theorem 3 (Information Multiplexing)**: When tasks $\{T_1, ..., T_k\}$ share data $D$, there exists information reuse:
$$I(D; \{T_1, ..., T_k\}) \leq \sum_i I(D; T_i) - I(D; T_i | T_j)$$

*Implication*: A joint optimization can reduce total information exposure.

## 4 IMFORM Framework

### 4.1 Framework Overview

```
┌────────────────────────────────────────────────────────────────────┐
│                         IMFORM Framework                           │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Raw Data D ──→ Task Analyzer ──→ Field Selector                  │
│                         ↓                                          │
│                Task T_i requires fields F_i                        │
│                         ↓                                          │
│                Privacy Estimator                                  │
│                I(F_i; PII) ≤ δ?                                  │
│                         ↓                                          │
│                Information Filter F_i(D) ──→ LLM API              │
│                         ↓                                          │
│                Task Output Y_i                                    │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

### 4.2 Core Components

**Task Analyzer**: Analyzes task $T_i$ to identify required information fields $F_i$.

**Privacy Estimator**: Computes privacy leakage $I(F_i; PII)$.

**Field Selector**: Selects the Minimum Sufficient Set $M_i$.

**LLM Interface**: Manages API calls with filtered input.

### 4.3 Three Implementations

| Implementation | Description | Use Case |
|----------------|-------------|----------|
| **Rule-based** | Expert-defined field selection rules | High-stakes, deterministic |
| **Learning-based** | Train a selector network | Large-scale, adaptive |
| **LLM-based** | Use LLM to self-select $M_i$ | General-purpose, flexible |

## 2 问题形式化

### 2.1 最小充分集（MSS）

**定义**：对于任务$T_i$和数据$D$，$M_i \subseteq D$是MSS当且仅当：
1. $Q(M_i, T_i) \geq Q(D, T_i) - \epsilon$
2. $|M_i| \ll |D|$

### 2.2 信息最小化原则

**假说1（IMP）**：对于大多数NLP任务，存在MSS $M_i$使得：
$$Q(M_i, T_i) \approx Q(D, T_i) \quad \text{且} \quad |M_i| \ll |D|$$

### 2.3 隐私-质量权衡

**定义**：隐私泄露$I(M_i; PII)$度量选中信息与敏感属性的互信息。

**假说2（不可能三角）**：无法同时达到：
- 零泄露：$I(M_i; PII) = 0$
- 最优质量：$Q(M_i, T_i) = Q^*(T_i)$
- 基线效率：$C(M_i) \leq C_{baseline}$

---

## 3 理论分析

### 3.1 为什么IMP有效

**定理1（任务无关性）**：对于任务$T_i$，存在$D \setminus M_i$与$T_i$几乎无关：
$$I(T_i; D \setminus M_i | M_i) \approx 0$$

### 3.2 信息压缩边界

**定理2（压缩率）**：任务$T_i$的最小压缩率$r_i = |M_i|/|D|$满足：
$$r_i \geq \frac{H(T_i | D \setminus M_i)}{H(D)}$$

### 3.3 多任务信息复用

**定理3（信息复用）**：当任务$\{T_1, ..., T_k\}$共享数据$D$时，存在信息复用空间。
